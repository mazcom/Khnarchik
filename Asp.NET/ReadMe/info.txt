REST - это архитектура, описывающая универсальный способ обработки и передачи состояний ресурсов по HTTP.

Одна сущность - 4 запроса:
GET
POST
PUT
DELETE

Плюсы такого подхода:
-Общность интерфейсов - рдин принцип работы дл каждой точки входа
-Масшатибируемость системы
-Промежуточный слой, добавлющий модульность и безопасность приложению.


OWIN & Katana

что такое OWIN - openWeb Interface for .NET - спецификаци дл абстракции web-приложени от сервера
OWIN выполнет 2 цели:
-Разработка новых компонент должна быть простой
-Приложение можно было бы легко перенести между компьютерами.

 
OWIN - это прежде всего абстракция сервера(Hosting Abstraction)
OWIN состоит из двух компонентов: 
-Словаря ключей IDICTIONARY<string, object>
-Интерфейс Func<Idictionary<string, object>, Task> - принимающий этот словарь и обробатывающий его
 Вся информация такая как заголовок, тело, типы всё в славере передается 
 

Katana- фреймоворк, разработанный коммандой .Net содержащий набор компонентов для создания и запуска вэб-приложений на базе OWIN

содержит в себе реализацию двух основых технологий:
-WebApi - вэб интерфейс на базе RESTful Api
-SignalR(webSockets) подключение на основе технологии веб сокетс

Swagger(OpenAPI) - спецификация для описания REST API или документация вашего API.
добавляется через пакет.
И потом можно будет открыть страничку на сайте по роуту: swagger/index.html и мы увидим все контроллеры и роуты.


HTTP - протокол прикладного уровня передачи гипертекстовых документов в формате HTML.
в настоящее время используется для передачи произвольных данных.  

Web Sockets - протокол связи поверх TCP- соединения, предназначенный для обмена сообщениями между браузером и веб-сервером
без потери производительности и соединения.
Здесь инициировать сообщение может буть то сервере будь то клиент. 

SignalR- - это библиотека от макрософт для работы с web-sockets на клинтский и серверной части.
